### 2.2 重点强化内容

根据客户需求，特别强化以下环节：

#### 🔴 重点 1：混合云场景下的异常检测
- **场景描述：** 公有云 + 私有云指标不一致，难以统一阈值
- **解决方案：** 
  - 使用 Isolation Forest 进行多维指标联合检测（Lab 2 增强版）
  - 基于 LSTM 自动编码器的跨云数据对齐（Lab 3 扩展）
- **实战演练：** 使用客户提供的跨云监控数据进行实验

#### 🔴 重点 2：MCP 统一模型服务与 Agent Skill 开发
- **场景描述：** 同时使用阿里云百炼、DeepSeek 等多个模型服务，接口不统一；运维工具分散，需要灵活组合调用
- **解决方案：**
  - 基于 MCP 协议构建统一的模型服务层，实现热切换和负载均衡
  - 开发标准化的 Agent Skill 插件（告警查询、日志分析、故障处置等）
  - 通过自然语言编排 Skill 形成自动化运维流程
- **实战演练：** 
  - 搭建 MCP Server，接入客户使用的云厂商模型
  - 分组开发 3-5 个定制化运维 Skill，集成到 Agent 系统

#### 🔴 重点 3：LLM 驱动的根因分析
- **场景描述：** 故障定位依赖人工经验，新人上手慢
- **解决方案：**
  - 构建基于 RAG 的故障知识库（Lab 6）
  - 训练 LLM 学习历史故障处理记录
  - 自然语言问答式故障诊断
  - **基于 MCP 提供的统一模型接口，快速切换不同 LLM 服务**
- **实战演练：** 导入客户历史故障文档，现场构建知识库

#### 🔴 重点 4：告警降噪与关联
- **场景描述：** 每日告警量过大，运维人员麻木
- **解决方案：**
  - 基于时间窗口的告警聚合
  - 基于语义聚类的告警分组
  - 动态阈值调整
- **实战演练：** 接入客户真实告警流，现场演示降噪效果

- 共享 Prometheus 服务器：1 台（8 Core / 16GB RAM）

| LLM API 配额：[X] QPS（根据并发测算）     |

### 3.3 数据来源

- **方案 A（推荐）：** 使用客户提供的脱敏生产数据
  - 优点：真实场景，代入感强
  - 缺点：需提前清洗和脱敏
  
- **方案 B：** 基于客户场景生成模拟数据
  - 优点：可控性强，无合规风险
  - 缺点：真实性略差

- **方案 C：** 完全使用标准数据集
  - 优点：开箱即用
  - 缺点：与客户场景有差距

**建议选择：方案 A + B 结合**
- 核心场景使用客户真实数据
- 补充场景使用模拟数据
