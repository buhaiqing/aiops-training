## 🎯 学习前的准备

### 1. 知识储备

#### 必需基础
- ✅ **Python 编程** - 能独立编写函数和类
- ✅ **Linux 基础** - 熟悉基本命令行操作
- ✅ **数学基础** - 了解概率统计和线性代数

- ⭐ 机器学习基础（知道什么是训练、预测）
- ⭐ 运维基础知识（了解监控、告警、日志）
- ⭐ 深度学习概念（知道神经网络是什么）

### 2. 环境准备

#### 最小化环境（推荐新手）
```bash
# 仅需 Python 3.10+
python3 --version  # 应 >= 3.10

# 无需安装任何包，使用系统 Python 即可
```

#### 完整环境（推荐有经验的学员）
```bash

curl -LsSf https://astral.sh/uv/install.sh | sh

# 或使用 venv
python3 -m venv aiops_env
source aiops_env/bin/activate

## 📚 学习方法论

### 五步学习法

```
1️⃣ 预习 → 2️⃣ 实践 → 3️⃣ 思考 → 4️⃣ 总结 → 5️⃣ 分享
```

#### Step 1: 预习（15 分钟）

在运行代码之前：

1. **阅读 README.md** - 了解实验目标
2. **浏览 lab_guide.md** - 理解技术原理
3. **查看数据文件** - 知道输入是什么
4. **预测输出** - 猜猜会得到什么结果

**检查清单**:
- [ ] 我知道这个实验要解决什么问题
- [ ] 我理解了核心算法的基本思想
- [ ] 我知道大概需要多长时间

#### Step 2: 实践（45-60 分钟）

动手运行代码：

1. **按步骤执行** - 严格按照指导书操作
2. **记录输出** - 保存关键日志和图表
3. **对比预期** - 实际结果和你想的一样吗？
4. **尝试修改** - 改变参数，观察变化

**检查清单**:
- [ ] 代码成功运行无错误
- [ ] 生成了预期的输出文件
- [ ] 我能解释每个步骤的作用
- [ ] 我尝试了至少一种参数变化

#### Step 3: 思考（20 分钟）

深入理解原理：

1. **为什么这样设计？** - 算法的设计思想
2. **优缺点是什么？** - 方法的局限性
3. **还能怎么用？** - 其他应用场景
4. **如何改进？** - 你的优化想法

**思考题示例**（以 Lab 1 为例）:
- 为什么选择 3σ而不是 2σ或 4σ？
- 如果数据不是正态分布怎么办？
- 滑动窗口大小对结果有什么影响？

**检查清单**:
- [ ] 我能用自己的话解释算法原理
- [ ] 我知道这个方法适合什么场景
- [ ] 我知道什么时候不应该用这个方法

#### Step 4: 总结（15 分钟）

整理学习笔记：

1. **核心概念** - 记录关键术语和定义
2. **代码片段** - 收藏有用的代码
3. **常见错误** - 避免下次踩坑
4. **知识联系** - 和其他 Lab 的关联

**笔记模板**:
```markdown
# Lab X: [名称]

## 核心概念
- 概念 1: ...
- 概念 2: ...

## 关键代码
```python
# 最有价值的代码片段
```

## 常见错误
1. 错误：...
   原因：...
   解决：...

## 与其他 Lab 的联系
- 与 Lab Y 的相似点：...
- 不同点：...
```

**检查清单**:
- [ ] 我完成了学习笔记
- [ ] 我记录了最重要的 3 个收获
- [ ] 我标记了还不理解的问题

#### Step 5: 分享（选做，额外 30 分钟）

教是最好的学：

1. **写技术博客** - 分享你的理解
2. **代码 review** - 给同学提建议
3. **回答问题** - 帮助其他学员
4. **改进项目** - 提交 PR

**检查清单**:
- [ ] 我在 Discussion 中发了帖子
- [ ] 我回答了至少一个问题
- [ ] 我考虑过提交 PR

## 💡 各 Lab 学习要点

### Lab 1: 3-Sigma 异常检测

**重点**:
- 理解正态分布和 3σ原则
- 掌握滚动均值和标准差的计算
- 学会动态阈值的设定

**难点**:
- 滑动窗口的选择和实现
- 非平稳序列的处理

**常见问题**:
- Q: 为什么我的数据不符合正态分布？
- A: 可以使用 Box-Cox 变换，或改用其他方法

**扩展思考**:
- 如何自适应调整窗口大小？
- 如何处理季节性数据？

---

### Lab 2: Isolation Forest

**重点**:
- 理解"孤立"的思想
- 掌握多维特征的联合检测
- 学会 contamination 参数调优

**难点**:
- 高维数据的可视化
- 异常得分的计算和解释

**常见问题**:
- Q: 如何选择合适的异常比例？
- A: 基于业务经验，或用网格搜索尝试不同值

**扩展思考**:
- 如何解释哪些特征导致了异常？
- 如何处理类别不平衡？

---

### Lab 3: LSTM Autoencoder

**重点**:
- 理解 LSTM 的门控机制
- 掌握自动编码器的重构误差
- 学会时序数据的滑动窗口处理

**难点**:
- 超参数调优（sequence_length, hidden_size）
- 防止过拟合

**常见问题**:
- Q: 训练损失不下降怎么办？
- A: 检查学习率、增加训练轮次、调整网络结构

**扩展思考**:
- 如何实现在线学习？
- 如何处理变长序列？

---

### Lab 4: Prophet 时序预测

**重点**:
- 理解时序分解（趋势 + 季节 + 节假日）
- 掌握置信区间的设置
- 学会残差分析

**难点**:
- 变点检测和调整
- 多重季节性的处理

**常见问题**:
- Q: Prophet 和 LSTM 应该选哪个？
- A: 强季节性选 Prophet，复杂模式选 LSTM

**扩展思考**:
- 如何加入外部回归变量？
- 如何处理突发事件的影响？

---

### Lab 5: One-Class SVM（开发中）

**重点关注**:
- 边界检测思想
- 核函数的选择
- 小样本场景的应用

---

### Lab 6: RAG 知识问答

**重点**:
- 理解 RAG 架构
- 掌握文本嵌入技术
- 学会向量相似度搜索

**难点**:
- 嵌入模型的选择
- 检索效率和准确率的平衡

**常见问题**:
- Q: 为什么要用双模式（高级 + 简化）？
- A: 兼顾教学效果和生产可用性

**扩展思考**:
- 如何实现多轮对话？
- 如何评估检索质量？

---

### Lab 7: LLM Agent 自主运维

**重点**:
- 理解告警关联分析
- 掌握根因推断方法
- 学会处置方案生成

**难点**:
- 规则引擎的设计
- 置信度的量化

**常见问题**:
- Q: 如何接入真实的 LLM？
- A: 参考 lab_guide.md 中的扩展任务

**扩展思考**:
- 如何实现多 Agent 协作？
- 如何从历史案例中学习？

---

### Lab 8: ChatOps 集成（规划中）

**重点关注**:
- Webhook 集成方法
- 自然语言生成
- 交互式对话设计

## 🛠️ 实用技巧

### 调试技巧

1. **打印中间变量**
   ```python
   print(f"Step 1: {data.shape}")
   print(f"Step 2: {result.mean():.4f}")
   ```

2. **使用断点调试**
   ```python
   import pdb; pdb.set_trace()  # 在关键位置暂停
   ```

3. **分步验证**
   ```bash
   python3 -c "from main import *; data = load_data(); print(data.head())"
   ```

### 性能优化

1. **向量化操作**
   ```python
   # ❌ 慢
   for i in range(len(data)):
       result[i] = data[i] * 2
   
   # ✅ 快
   result = data * 2
   ```

2. **批量处理**
   ```python
   # 分批处理大数据集
   batch_size = 1000
   for i in range(0, len(data), batch_size):
       process(data[i:i+batch_size])
   ```

3. **缓存中间结果**
   ```python
   from functools import lru_cache
   
   @lru_cache(maxsize=None)
   def expensive_computation(x):
       return x ** 2
   ```
