# AIOps 培训前访谈提纲

## 📞 访谈基本信息

**访谈目的：** 基于客户填写的调研问卷，澄清关键细节，确保培训方案精准匹配需求

**建议时长：** 60-90 分钟

**参会人员：**
- 客户方：培训负责人、运维经理、技术骨干（2-4 人）
- 培训方：主讲讲师、课程顾问（1-2 人）

---

## 一、开场与背景了解（10 分钟）

### 1.1 破冰与介绍
- 双方团队自我介绍
- 简述访谈目的和流程
- 确认客户已填写调研问卷

### 1.2 项目背景深挖
**关键问题：**
1. "能否介绍一下发起这次培训的初衷？是遇到了什么具体的痛点或挑战？"
2. "管理层对这次培训的期望是什么？希望解决什么问题？"
3. "除了问卷中提到的需求，还有哪些未明说的隐性需求？"

**观察点：**
- 培训是主动需求还是被动任务？
- 是否有明确的业务驱动因素（如近期故障频发、数字化转型压力等）？
- 决策层的关注点 vs 执行层的关注点是否有差异？

---

## 二、学员情况澄清（15 分钟）

### 2.1 技术基线确认
**讨论要点：**
- 根据问卷反馈，确认学员的 Python/ML/AI 基础水平
- 是否存在两极分化？是否需要分班教学？
- 是否有特殊背景的学员（如从开发转运维、从传统运维转型等）？

**关键问题：**
1. "问卷中提到学员 Python 水平是 [入门级]，能否举例说明他们目前能做什么、不能做什么？"
2. "团队中是否有完全没有编程经验的成员？如果有，比例多少？"
3. "之前是否组织过类似的 AI/ML 培训？效果如何？"
4. "学员们最担心跟不上哪部分内容？"

### 2.2 学习动机与阻力
**关键问题：**
1. "学员们对这次培训的积极性如何？是主动想学还是被要求参加？"
2. "他们最希望通过培训获得什么？（晋升、转岗、解决工作问题等）"
3. "是否存在抵触情绪？比如担心 AI 会取代自己的工作？"

**应对策略：**
- 如有抵触情绪，需在培训开场强调"AI 是工具，不是替代者"
- 强调 AIOps 是提升效率，让运维人员从事更有价值的工作

---

## 三、业务场景深度挖掘（20 分钟） ⭐ 核心环节

### 3.1 混合云架构细节
**讨论要点：**
- 画出客户的混合云架构图（可提前准备草图）
- 标注各云平台的职责分工
- 识别数据流和调用链

**关键问题：**
1. "能否详细描述一下当前的混合云架构？哪些业务跑在公有云，哪些在私有云？"
2. "为什么选择这样的架构？是出于成本、合规还是其他考虑？"
3. "跨云的网络是如何打通的？专线/VPN/公网？"
4. "监控数据是如何采集的？每个云独立监控还是有统一平台？"
5. "遇到的最大运维挑战是什么？请举 1-2 个具体例子。"

**追问技巧：**
- "能再具体一点吗？比如上次发生这种情况是什么时候？"
- "当时是如何处理的？花了多长时间？"
- "如果重新来一次，你们会怎么处理？"

### 3.2 典型故障案例复盘
**讨论要点：**
- 选取 2-3 个最近发生的典型故障
- 按照时间线还原处理过程
- 识别痛点和改进空间

**关键问题：**
1. "能否分享一个印象最深的故障案例？从发现到恢复的全过程。"
2. "这个故障的根本原因是什么？是如何定位到的？"
3. "整个处理过程中，哪个环节最耗时？哪个环节最容易出错？"
4. "如果有 AIOps 工具，您觉得哪个环节可以被优化？"

**示例引导：**
- "比如 CPU 飙升的场景，通常是如何发现的？是用户投诉还是监控告警？"
- "告警响起后，第一步做什么？如何判断是扩容还是重启还是其他操作？"
- "有没有遇到过告警太多麻木了，结果漏掉了重要告警的情况？"

### 3.3 现有工具链评估
**讨论要点：**
- 列出当前使用的所有监控/运维工具
- 评估每个工具的满意度
- 识别集成痛点

**关键问题：**
1. "目前使用的监控工具中，最满意的是哪个？为什么？"
2. "最不满意的又是哪个？有哪些槽点？"
3. "这些工具之间是否能联动？比如监控发现异常后能否自动触发工单？"
4. "是否有想过整合这些工具？难点在哪里？"

---

## 四、技术环境验证（10 分钟）

### 4.1 实验环境可行性
**讨论要点：**
- 确认培训环境与客户环境的兼容性
- 评估数据采集的可行性
- 识别潜在的技术障碍

**关键问题：**
1. "培训时是否方便搭建与客户环境类似的实验环境？"
2. "能否提供脱敏后的真实监控数据用于实验？数据量大概多大？"
3. "如果不能提供真实数据，我们基于您的场景定制模拟数据，是否可以？"
4. "培训中的 LLM 服务，是希望用公有云 API 还是本地部署？"
   - 如选本地：确认 GPU 资源是否充足
   - 如选公有云：确认网络和安全合规是否允许

### 4.2 集成需求确认
**关键问题：**
1. "是否希望在培训中演示与客户现有系统的集成？比如读取 Prometheus 数据。"
2. "是否需要对接现有的告警渠道？（钉钉/企微/邮件等）"
3. "是否有 ITSM 工单系统需要对接？（Jira/ServiceNow 等）"

**技术预研：**
- 如需集成，需提前获取 API 文档和测试环境权限
- 评估集成复杂度，必要时安排额外时间

---

## 五、解决方案选型探讨（15 分钟）

### 5.1 采购策略澄清
**讨论要点：**
- 理解客户的决策流程
- 识别关键决策因素
- 评估预算约束

**关键问题：**
1. "关于 AIOps 建设，目前是倾向于自建、采购还是混合模式？"
2. "如果是采购，决策流程是怎样的？需要哪些部门参与？"
3. "预算审批的难度如何？更关注一次性投入还是持续成本？"
4. "在选择方案时，最看重的 3 个因素是什么？（功能、成本、易用性、服务等）"

### 5.2 竞品对比期望
**讨论要点：**
- 客户已经接触过哪些供应商？
- 对哪些方案有好感或疑虑？
- 需要提供什么样的对比分析？

**关键问题：**
1. "目前是否已经接触过一些 AIOps 供应商？印象如何？"
2. "是否有已经在使用的商业产品？体验如何？"
3. "您希望我们在培训中提供哪些产品的对比分析？"
4. "对于开源方案，最大的顾虑是什么？（技术支持、学习成本、稳定性等）"

**准备材料：**
- 商业产品对比矩阵（功能、价格、优劣势）
- TCO（总体拥有成本）计算模型
- 典型客户案例参考

## 六、培训细节确认（15 分钟）

### 6.1 内容深度与节奏
**讨论要点：**
- 根据学员基线调整内容难度
- 确认重点章节和时间分配
- 识别需要特别强化的部分

**关键问题：**
1. "根据我们的课程模块，您最感兴趣的是哪几个 Lab？为什么？"
2. "哪些内容是您认为必须覆盖的？哪些可以简略？"
3. "对于 LLM 应用开发（RAG/Agent），您希望深入到什么程度？"
   - 了解原理和使用
   - 能独立开发和调优
   - 能修改源码和定制

**方案调整建议：**
- 如学员基础弱：增加 Day 0 Python 补课环节
- 如关注混合云：强化 Lab 的多云场景配置
- 如需选型建议：增加"解决方案对比"专题分享

### 6.2 实验形式偏好
**关键问题：**
1. "更喜欢哪种实验形式？"
   - 完全 guided（按步骤操作）
   - 半开放（给定目标，自行设计）
   - 完全开放（自由发挥）
2. "是否需要分组竞赛？设置奖项激励？"
3. "实验数据希望用："
   - 模拟数据（干净、可控）
   - 定制场景数据（贴近实际）
   - 真实脱敏数据（最有代入感）

### 6.3 考核与认证
**关键问题：**
1. "是否需要结业考试或认证？"
2. "是否希望颁发结业证书？需要什么级别的认证？"
3. "培训后是否希望有持续的技能跟踪？比如 3 个月后回访测评？"

## 七、成功标准与期望管理（10 分钟）

### 7.1 短期效果期望
**关键问题：**
1. "培训结束后，您希望学员能够立即做到什么？"
   - 独立搭建异常检测系统
   - 理解 AIOps 的基本思路
   - 能评估和选型产品
2. "如何定义这次培训是成功的？有什么可衡量的指标吗？"

### 7.2 长期价值期望
**关键问题：**
1. "3 个月后，您希望看到什么变化？"
   - MTTR 降低 X%
   - 误报率降低 Y%
   - 团队能自主开发 AIOps 工具
2. "这次培训在团队整体能力建设中的定位是什么？"
   - 启蒙课（了解概念）
   - 入门课（掌握基础技能）
   - 进阶课（深度学习前沿技术）

### 7.3 风险与约束
**关键问题：**
1. "您觉得这次培训可能遇到的最大挑战是什么？"
   - 学员基础参差不齐
   - 时间不够用
   - 内容太难跟不上
   - 培训后无法落地
2. "有哪些是我们需要提前规避的风险？"

## 💡 访谈技巧提示

### 挖掘真实需求的 SPIN 法则
- **S（Situation）**：了解现状 → "目前是如何处理 XX 问题的？"
- **P（Problem）**：发现问题 → "这样做会遇到什么困难？"
- **I（Implication）**：放大影响 → "这个问题会导致什么后果？"
- **N（Need-payoff）**：引导需求 → "如果有 XX 方案，能解决这个问题吗？"

### 处理敏感问题的技巧
- **预算问题**：不要直接问"有多少钱"，而是问"倾向什么价位的方案"
- **竞争产品**：不要贬低竞品，而是问"您觉得 XX 产品怎么样"
- **技术短板**：不要说"你们这个不行"，而是说"可以考虑升级到 XX"

### 识别决策信号
**积极信号：**
- 开始讨论具体时间和人员安排
- 询问价格和合同细节
- 邀请更多相关人员参会
- 提出定制化需求

**犹豫信号：**
- 反复强调"再看看"、"再比较"
- 对培训效果表示怀疑
- 不承诺具体时间
- 需要向上级汇报
