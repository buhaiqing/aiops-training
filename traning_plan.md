# AI 运维培训整体方案

## 一、培训目标

培养具备实际 AI 运维能力的专业人才，重点掌握：
- **智能分析**：利用 AI 技术进行运维数据分析
- **根因分析**：快速定位系统故障根本原因
- **智能检测**：异常检测与预警
- **应急处理**：智能化应急响应与恢复
- **LLM 应用**：大模型在 AIOps 中的实战应用

- ✅ 配置基础告警规则
- ✅ 部署本地 LLM 模型（Llama/Qwen）

### 模块 2：智能异常检测 (30% - 重点)

**目标**：掌握多种异常检测方法并实际应用

**内容**：

#### 1. 统计方法实战
- 3-Sigma 原则检测 CPU/内存异常
- 移动平均线检测流量波动
- 分位数检测响应时间异常

**动手实验**: 详见 `Labs/lab1_3sigma_anomaly_detection/README.md`

#### 2. 机器学习方法
- Isolation Forest 检测多维指标异常
- One-Class SVM 识别未知故障模式
- DBSCAN 聚类发现异常群体

**动手实验**: 详见 `Labs/lab2_isolation_forest/README.md`

#### 3. 深度学习方法
- LSTM 自动编码器检测时序异常
- Transformer 预测异常趋势
- 无监督学习降低标注成本

**动手实验**: 详见 `Labs/lab3_lstm_autoencoder/README.md`

#### 4. Prophet 时序预测与异常检测 ⭐
- Prophet 模型原理与实战
- 趋势项、季节项、节假日分解
- 基于预测残差的异常检测
- 多场景时序预测应用

**动手实验**: 详见 `Labs/lab4_prophet_forecast/README.md` ✅ 已完成

#### 5. One-Class SVM 未知故障模式识别
- 支持向量机理论
- 核函数技巧
- 边界检测方法
- 小样本异常检测

**动手实验**: 详见 `Labs/lab5_ocsvm_anomaly_detection/README.md` 🚧 开发中

#### 6. LLM 增强方法 ⭐
- 使用 LLM 生成异常检测规则
- 基于 LLM 的日志语义异常识别
- 多模态异常检测（指标 + 日志联合分析）

**实操项目**：
- ✅ 构建 CPU 使用率异常检测器（准确率>90%）  
- ✅ 实现网络流量突发检测系统  
- ✅ 开发多指标联合异常检测平台  
- ✅ 使用 LLM 自动分析日志异常模式
- ✅ 基于 LLM 的告警描述自动生成

### 模块 3：智能根因分析 (30% - 核心)

**目标**：快速定位故障源头，缩短 MTTR

**内容**：

#### 1. 拓扑分析方法
- 服务依赖图构建
- 调用链追踪（SkyWalking/Jaeger）
- 基于图的根因传播分析

#### 2. 相关性分析
- Pearson/Spearman 相关系数计算
- Granger 因果检验
- 贝叶斯网络推断

#### 3. 决策树方法
- 随机森林特征重要性排序
- XGBoost 根因评分
- SHAP 值解释模型决策

#### 3. LLM 驱动的根因分析 ⭐

- **RAG 知识库构建**：历史故障文档向量化 ✅ 详见 Lab 6
- **智能问答系统**：自然语言查询故障原因
- **日志语义理解**：LLM 解析错误日志含义
- **多源数据融合**：LLM 关联指标、日志、链路数据
- **推理链生成**：LLM 输出根因分析推理过程
- **Few-shot Learning**：示例学习提升诊断准确性

#### 4. 知识图谱
- 构建运维知识库
- 历史故障匹配
- 推荐解决方案

**实操项目**：
- ✅ 实现微服务调用链根因定位  
- ✅ 开发多维度指标关联分析工具  
- ✅ 构建基于 RAG 的故障知识问答系统
- ✅ 使用 LLM 自动生成根因分析报告
- ✅ LLM + 知识图谱联合推理故障原因
- ✅ 模拟真实故障进行根因分析演练

### 模块 4：智能告警与应急处理 (30% - 关键)

**目标**：减少告警风暴，自动化应急响应

**内容**：

#### 1. 告警降噪
- 告警聚合（时间/空间/语义）
- 告警压缩与去重
- 动态阈值调整

#### 2. 告警关联
- 基于规则的关联
- 基于机器学习的关联挖掘
- 时序模式匹配

#### 3. LLM 智能告警处理 ⭐

- **告警语义聚类**：LLM 理解告警含义后分组
- **自然语言告警摘要**：自动生成人类可读的告警总结
- **智能告警路由**：LLM 判断告警类型并分派给对应人员
- **告警优先级评估**：LLM 结合上下文判断严重程度
- **应急预案生成**：LLM 根据故障描述自动生成处置步骤
- **ChatOps 集成**：通过对话机器人执行应急操作 🚧 规划中 (Lab 8)

#### 4. 自动化应急
- 预案自动生成
- 智能决策建议
- 自动化修复脚本执行
- 回滚机制设计

#### 5. Agent 自主运维 ⭐ ✅ 详见 Lab 7

- LLM Agent 自主诊断流程
- Tool Learning：教会 LLM 使用运维工具
- 自主执行健康检查、重启、扩容等操作
- Human-in-the-loop：人工确认关键操作

**实操项目**：
- ✅ 实现告警风暴抑制系统（减少 80% 冗余告警）  
- ✅ 开发告警关联分析引擎  
- ✅ 构建 LLM 告警助手（支持自然语言问答）
- ✅ 使用 LLM 自动生成应急预案
- ✅ 开发 ChatOps 机器人（钉钉/企微/飞书集成）
- ✅ LLM Agent 自主故障诊断与修复演练
- ✅ 模拟故障进行应急演练（从检测到恢复全流程）

## 四、教学方法

### 7:2:1 时间分配
- **70% 动手实操**：每节课至少 2 个代码项目
- **20% 案例分析**：拆解真实生产事故
- **10% 要点讲解**：只讲关键思路和避坑指南

### 项目驱动学习
每个知识点配套：
- 📝 数据集（公开数据集 + 脱敏生产数据）
- 💻 代码模板（Jupyter Notebook）
- 🎯 练习任务（明确验收标准）
- ✅ 参考答案（最优实践）

### 渐进式难度
```
基础操作 → 单一算法 → 组合应用 → 综合系统
```
